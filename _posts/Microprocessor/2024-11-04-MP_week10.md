### 2. Condition Code
- `EFLAGS` 레지스터  
![x86_condition_code](TODO)
  - `OF` : Overflow Flag
  - `IF` : Interrupt Enable Flag
  - `SF` : Sign Flag
  - `ZF` : Zero Flag
  - `CF` : Carry Flag

- 이 외에도 많은 플래그들이 존재한다.  

<br>

### 3. 명령어
- 명령어 구조
  - 명령어 당 `1` ~ `15` byte
  - Prefix `0` ~ `4` byte
  - Opcode `1` ~ `2` byte (명령어에 따라서 ModReg의 일부도 사용)
  - ModReg `1` byte
  - IndBase `1` byte (Scale, Index, Base)
  - Displacement `0` ~ `4` byte
  - Immediate `0` ~ `4` byte  

  <br>

- 명령어 집합
  - 2 Operand  형식  
  `addl %ebx, %eax` : `%eax <- %eax + %ebx`  

  - 스택에 대한 push / pop 명령어 제공  
  `pushl %ebp`  
  `popl %eax`  

  - 다양한 명령어들에 메모리 접근 기능 포함  
  `negl variable`  
  `addl 4(%edi), %edx`  
  `addl $3, 0x2000`  

  - MUL, DIV, Floating point 연산, ...etc  

  - 총 300여 개의 명령어 (Pentinum 기준)
