### 2. 보안과 권한
- 오라클 multi-tenant 아키텍처  
![oracle_multi_tenant_architecture][def2]  

- 오라클 DBMS 접근권한  
![oracle_dbms_access_rights][def3]  

<br>

- 테이블스페이스 생성 (10MB 크기)  

```sql
CREATE TABLESPACE [테이블스페이스_이름]
  DATAFILE '파일_경로' SIZE 10M;
```

- 테이블스페이스 삭제 (데이터 파일까지 포함하여 모두 삭제)  

```sql
DROP TABLESPACE [테이블스페이스_이름]
  INCLUDING CONTENTS AND DATAFILES;
```

<br>

- 사용자 계정 생성  

```sql
CREATE USER [사용자_이름]
  IDENTIFIED BY [비밀번호]
  DEFAULT TABLESPACE [테이블스페이스_이름];
```

- 사용자 계정 설정 변경  

```sql
ALTER USER [사용자_이름]
  IDENTIFIED BY [새_비밀번호];
```

- 사용자 계정 삭제 (사용하는 모든 테이블도 삭제)  

```sql
DROP USER [사용자_이름] CASCADE;
```

<br>

- 권한 부여 (`CONNECT`, `RESOURCE`, `UNLIMITED TABLESPACE` 권한을 부여)  

```sql
GRANT CONNECT, RESOURCE, UNLIMITED TABLESPACE TO [사용자_이름];  
```

- 권한 부여 (특정 테이블에 대한 `SELECT` 권한을 부여)  

```sql
GRANT SELECT ON [테이블_이름] TO [사용자_이름];
```

- ★ 권한 부여 (특정 테이블에 대한 `SELECT`, `UPDATE` 권한을 `WITH GRANT OPTION`과 함께 부여)  

```sql
GRANT SELECT, UPDATE ON [테이블_이름] TO [사용자_이름] WITH GRANT OPTION;
```

- 권한 부여 (특정 두 개의 테이블에 대한 `SELECT` 권한을 부여)  

```sql
GRANT SELECT ON [테이블_이름1], [테이블_이름2] TO [사용자_이름];
```

- 권한 부여 (특정 테이블을 모든 사용자가 `SELECT`할 수 있도록 권한을 부여)  

```sql
GRANT SELECT ON [테이블_이름] TO PUBLIC;
```

<br>

- 롤(ROLE) : 데이터베이스 객체에 대한 권한을 모아둔 집합
  - 역할 생성  

  ```sql
  CREATE ROLE [역할_이름];
  ```

  - 역할 제거  

  ```sql
  DROP ROLE [역할_이름];
  ```

  - 역할에 권한 부여  

  ```sql
  GRANT [권한_종류] ON [테이블_이름] TO [역할_이름];
  ```

  - 역할에 권한 회수

  ```sql
  REVOKE [권한_종류] ON [테이블_이름] FROM [역할_이름];
  ```

  - 사용자에게 역할 부여  

  ```sql
  GRANT [역할_이름] TO [사용자_이름];
  ```

  <br>

### 3. 백업과 복원
- 백업(backup) : 데이터베이스에서 예상하지 못한 장애에 대비하여 데이터베이스를 복제하여 보관하는 작업

- 복원(recovery) : 장애가 발생하여 운영 중인 데이터에 손상이 발생했을 때 기존에 복사해 둔 백업 파일을 사용하여 원래대로 되돌려 놓는 작업
