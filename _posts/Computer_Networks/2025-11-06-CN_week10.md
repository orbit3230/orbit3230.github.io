## Information-centrix networking
### 1. Motivations
- Observation
  - TCP/IP based Internet  
  ![tcp_ip_internet_1][def]
  ![tcp_ip_internet_2][def2]
    - 1960년대에 디자인
    - End-to-end 커뮤니케이션 모델
    - Location 기반 식별자 : IP

  - Internet traffic trend  
  ![internet_traffic_trend][def3]  
    - 대량의 데이터
    - 중복된 데이터

  - Wired -> Wireless  
  ![wired_to_wireless][def4]  
    - Mobility가 매우 중요한 requirement  

  - Location(IP address)의 변화가 존재하던 연결을 끊음  
  ![mobility_issue][def5]  
    - Mobility 관리 오버헤드 문제가 제기됨  

<br>

- Claim
  - 기존이 IP 대신, 아래와 같은 새로운 네트워크 레이어로 전환하자
    - 데이터가 더 효율적으로 전달될 수 있는
    - Mobility가 더 쉽게 구현될 수 있는
    - 데이터 보안이 더 강화될 수 있는  
    - ...  
    ![new_network_layer][def6]  

<br>

- Information-Centric Networking (ICN)  
![icn_overview][def7]  
  - 추상적인 관점에서 보자면, 인터넷은 커다란 데이터베이스인 것이다.  
  ![internet_as_database_query][def8]  
  ![internet_as_database_response][def9]  

  - Receiver-driven communication
    - Requester가 커뮤니케이션을 시작
    - Queries(interests) / Data(responses)
    - TCP/IP 네트워킹에서의 sender-driven communication과 **대조적**

  - 컨텐츠는 location identifier(e.g., IP address)가 아닌 ID(e.g., name)으로 addressed

  - 네트워크 노드 어디서든 In-network 캐싱 가능
    - 라우터가 더 많은 일을 수행
    - TCP/IP 네트워킹에서의 라우팅/스위칭과 **대조적**  

<br>

### 2. ICN Architectures
- ICN proposals
  - Centralized architecture (유럽에서 제안)  
  ![centralized_icn_architecture][def10]  
    - Pub/sub 인터넷 라우팅 패러다임
    - Flat name 구조
    - edge 라우터에서 캐싱
    - Name Resolution System

  - Decentralized architecture (미국에서 제안)  
  ![decentralized_icn_architecture][def11]  
    - Named Data Networking (NDN)
    - Content-Centric Networking (CCN)  

<br>

- Named Data Networking (NDN)
  - 컨텐츠가 이름으로 식별되는 네트워크 아키텍처  
  - 컨텐츠 이름은 hierarchical structure를 가짐  
    - e.g. `Youtube/movies/soccer/Liverpool_vs_ManCity.mp4`

  - NDN 메시지
    - Query 패킷  
    ![ndn_interest_packet][def12]  
      - Interest라고 불림
      - Name : 식별자
      - Nonce : 이름으로 표시된 컨텐츠의 Uniqueness를 보장
      - Selector : 정보와 관련된 matching & security
      - Guider : 라우팅/포워딩과 관련

    - Data 패킷  
    ![ndn_data_packet][def13]  
      - Response라고 불림
      - Name : 식별자
      - MetaInfo
      - Content : 실제 데이터
      - Signature : 보안 메커니즘
