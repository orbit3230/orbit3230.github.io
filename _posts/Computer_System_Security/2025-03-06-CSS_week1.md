## Course Overview
- 컴퓨터 보안과 관련된 기초 지식과,  
컴퓨터 보안 관련 실무와 연결되는 여러 가지 개념 및 응용 방식들을 소개.  
더불어 국제적으로 인정받는 **CISSP**(Certified Information Systems Security Professional) 자격 시험에 포함된 내용을 다룸.  

<br>

## Computer Security Overview
### 1. 컴퓨터 보안 개념
- NIST(National Institute of Standards and Technology)에서는 컴퓨터 보안을 다음과 같이 정의한다.  

  > 자동화된 정보 시스템 내의 **자원**(하드웨어, 소프트웨어, 펌웨어, 정보/데이터, 통신)들의  
  **무결성**(Integrity), **기밀성**(Confidentiality), **가용성**(Availability)을 보존하기 위해 제공되는 보호  

  - 이 세 가지가 보안요구사항 CIA 3 원소이다.  
  ![cia][def]  

  - 주요 보안 개념/목표 (FIPS PUB 199)  
    - **기밀성** : 개인 프라이버시와 소유권 정보를 보호, 정보 접근 및 공개를 제한  
    - **무결성** : 정보의 부인 방지와 진위성을 보증, 변조 및 파괴로부터 보호  
    - **가용성** : 정보에 대한 적절하고 신뢰성 있는 접근과 사용을 보증  

  - 3 Level 보안 위반 영향 (FIPS PUB 199)  
    - 낮은(low) 레벨 : 손실 시 제한적인 피해
    - 보통(moderate) 레벨 : 손실 시 중대한 피해
    - 높은(high) 레벨 : 손실 시 치명적인 피해
     
     <br>

### 2. 컴퓨터 시스템 자산 (Resources)  
- 하드웨어 - 컴퓨터 시스템, 데이터 처리/저장/통신 장비
- 소프트웨어 - 운영체제, 시스템 도구, 응용 프로그램
- 데이터 - 파일, DB, 암호 파일 등 보안 관련 데이터  
- 네트워크 - LAN, WAN, 브릿지, 라우터 등

<br>

### 3. 위협 및 공격
- 시스템 자원의 취약성으로 인한 결과
  - 무결성 상실 -> 시스템의 오염(corruption)
  - 기밀성 상실 -> 시스템의 누수(leakage)
  - 가용성 상실 -> 시스템의 작동 불가 혹은 속도 저하  

|위협의 결과|위협행위(공격)|
|:---:|:---:|
|비인가 노출|노출, 도청, 간섭, 침입|
|기만|매스커레이드(Masquerade, 위장), 변조, 부인|
|분열|무력화, 오염, 방해|
|횡령|전용, 오용|

<br>

- 위협(Threat) : 취약성을 이용하여 시스템 자원을 공격하는 것  
  - 리소스에 잠재적인 보안 손실을 야기한다.  

- 공격의 형태  
  - **수동적(passive) 공격** - 시스템 정보를 이용/취득 하려는 시도. 시스템 리소스에는 영향 X  
    - 탐지는 어렵지만, **방지**는 쉬움 (암호화)  
    - 종류
      - 메세지 내용 노출
      - 트래픽 분석

  - **능동적(active) 공격** - 시스템 리소스를 변경하거나 시스템 동작에 영향을 미치려는 시도  
    - 예방은 어려우며, **탐지 후 복구**가 필요  
    - 종류
      - 지연(replay)
      - 매스커레이드(masquerade)
      - 메세지 변경(modification)
      - 서비스 거부(denial of service)

  - 내부(inside) 공격 - 보안 경계 내부에서의 공격  

  - 외부(outside) 공격 - 보안 경계 외부에서의 공격  

  <br>

### 4. 대응(Countermeasures)
- 보안 공격에 대응하는 수단
  - (1) 공격 방지 (Prevent)
  - (2) 공격 탐지 (Detect)
  - (3) 복구 (Recover)  

- 주의할 점 / 목표
  - 대응 수행 후에도 잔재 취약점이 존재할 수 있음.
  - 대응이 새로운 취약점을 유발할 수 있음.
  - 리소스에 대한 잔류 위험 수준을 ***최소화*** 시키는 것이 목표  

  <br>

### 5. 보안 설계 원칙  
- **메커니즘의 효율성** : 설계에 사용되는 HW/SW가 가능한 간단하고 규모가 작아야 함. (complicate -> more lacks)  

- **고장-안전 디폴트** : Acees에 대한 결정 여부에 대한 Default는 Deny.  

- **완전한 매개** : 모든 Access는 Access Control Mechanism을 통해 제어

- **개방적 설계** : 보안 메커니즘의 설계가 개방적이어야 함.  
(e.g., 암호화 알고리즘은 **공개**, 암호 키만 비공개)

- **권한 분리** : 중요한 리소스에 대해서는 Multiple Authentication 사용.  
(e.g., Multi-factor user authentication, 프로그램을 여러 부분으로 나누어 다른 권한 부여)  

- **최소 권한**(principle of least privilege) : 모든 프로세스와 시스템 사용자에게, 작업을 수행하는 데 있어 필요한 **최소한의 권한** 만을 부여  

- **최소한의 공통 메커니즘** : 설계 과정에서 사용자들 사이에 보안을 제공하기 위해, 다른 사용자들과 공유하는 기능을 최소화  

- **심리학적 수용성** : 보안 메커니즘이 사용자의 작업에 지나치게 간섭 X. 동시에 Access에 대한 권한 부여의 요구는 만족해야 함.  

- **격리** : 누구에게나 접근이 허용된 시스템을 중요한 리소스로 부터 격리.  
각 사용자의 프로세스/파일을 다른 사용으로부터 격리.  
보안 메커니즘은 이 메커니즘을 접근할 수 없도록 격리.  

- **캡슐화** : 객체 지향 관점에서의 격리. 프로시저(procedure)와 데이터 객체 집합을 캡슐화하여 보안성 향상  

- **모듈화** : 보안 기능들을 별도의 보호받는 모듈로 분리하는 모듈러 아키텍쳐를 사용  

- **계층화**(layering) : 다수의 중첩된 보호 수단을 사용하는 것. aka **심층 방어**(defense in depth). 

- **사용자 친화적** : 사용자가 프로그램이나 인터페이스를 이용하는 데 큰 어려움이 없어야 함.  

<br>

### 6. 공격 범위 (Attack Surface)
- 시스템에서 접근할 수 있고, 악용될 수 있는 취약점들.  
(e.g., 열린 포트(port), 방화벽 내부 서비스, 표준 데이터 교환 포맷을 처리하는 프로그램 코드, 인터페이스, SQL, 웹 폼, 사회 공학적 공격(real world)에 취약하고 민감한 정보에 접근할 수 있는 직원)  

<br>

- 공격 범위 분류
  - 네트워크 공격 범위
    - 네트워크, 인터넷 취약점을 통한 공격
    - 대표적으로 네트워크 프로토콜 취약점 (서비스 거부 공격, 통신 라인 붕괴)  

  - 소프트웨어 공격 범위
    - 애플리케이션, 유틸리티, OS 코드의 취약점
    - 웹 서버 소프트웨어 취약점에 특히 주목  

  - 인간 공격 범위
    - 사회 공학, 인간적 오류, 신뢰하던 내부 직원/외부인에 의해 생성된 취약점  

    <br>

### 7. 컴퓨터 보안 전략  
- 명세서/**정책** -> "하고자 하는 것이 무엇인가?"  
  - 시스템이나 조직이 중요한 시스템 리소스에 제공하는 보안 서비스 방법에 대한 일반적인 **규칙**, **관행**  
  - 고려해야 할 사항 : **자산의 가치**,  시스템의 **취약점**, 잠재적인 위협/공격 가능성 
  - 절출 사항 : 사용의 용이성 vs 보안 / 보안 비용 vs 오류 시 복구 비용  

- **구현**/메커니즘 -> "어떻게 하는가?"
  - 예방(Prevention)
  - 대응(Response)
  - 탐지(Detection)
  - 복구(Recovery)  

- **보증**/**평가** -> "잘 동작되고 있는가?"  
  - 보증 : 보안 수단의 **확신 정도**
  - 평가 : 테스트 및 formal한 분석/수학적 기법으로 평가
