## 기계 학습
### 1. 기계학습이란
#### [1] 기계 학습 개념
- 간단한 기계 학습 예제
  - 간단한 4개 점 데이터  
  ![simple_ml_example][def]  

- 예측(prediction, inference) 문제
  - 임의의시간이 주어지면, 이 때 이동체의 위치는?
  - 회귀(regression) 문제와 분류(classification) 문제로 나뉨
    - 회귀는 목표치가 실수, 분류는 목표치가 범주형  

<br>

- 원점을 지나면서 두 점 `(3, 2)`, `(2, 3)`을 나타내는 직선의 방정식은?  
![line_equation][def2]  
  - 각 점에서의 거리가 가장 짧은 직선  
  ![line_equation][def3]  

  - 미분  
  ![line_equation][def5]  

  - 학습  
  ![line_equation][def4]  

- 훈련집합(training set) / 학습집합(learning set)
  - 가로축은 특징(feature), 세로축은 목표치
  - 관측한 4개의 점이 훈련집합을 구성함  
  ![training_set][def6]  

<br>

- 데이터를 어떻게 모델링할 것인가
  - 직선 모델의 수식
    - 2개의 매개변수(parameter) `w`와 `b`  
    `y = wx + b`  

- 기계 학습은 
  - 가장 정확하게 예측할 수 있는, 즉 최적의 매개변수를 찾는 작업
  - 처음에는 임의의 값에서 시작하고, 점점 성능을 개선하여 최적에 도달
  - 데이터가 달라지면 매개변수도 당연히 달라진다.  

<br>

- 기계 학습의 궁극적인 목표  
  - 테스트(test) : 훈련 집합에 없는 '새로운' 샘플에 대한 목표값을 예측하는 과정
  - 테스트의 오류를 최소화하는 것이 목표
  - 테스트 집합에 대한 높은 성능을 **일반화(generalization)** 능력이라 부름  
  - 데이터베이스 : 훈련집합(training set + validation, development set) + 테스트 집합(test set)  

<br>

- 학습은 파라미터를 방향성에 맞춰 조금씩 수정해가는 과정  
![learning_process][def7]

### 2. 특징 공간에 대한 이해
#### [1] 1차원과 2차원 특징 공간
- 1차원 특징 공간  
![1d_feature_space][def8]  

- 2차원 특징 공간  
![2d_feature_space][def9]  
  - 특징 벡터 표기  
  `x = (x`<sub>`1`</sub>`, x`<sub>`2`</sub>`)`<sup>`T`</sup>  
  - 예시  
    - `x` = `(몸무게, 키)`<sup>`T`</sup>, `y` = `장타율`  
    - `x` = `(체온, 두통)`<sup>`T`</sup>, `y` = `감기 여부`  

<br>

#### [2] 다차원 특징 공간  
- 다차원 특징 공간 예제  
![nd_feature_space][def10]  

<br>

- d-차원 데이터  
  - 특징 벡터 표기  
  `x = (x`<sub>`1`</sub>`, x`<sub>`2`</sub>`, ..., x`<sub>`d`</sub>`)`<sup>`T`</sup>  

- d-차원 데이터를 위한 데이터를 위한 학습 모델  
  - 직선 모델을 사용하는 경우 매개변수 = `d + 1`  
  `y = w`<sub>`1`</sub>`x`<sub>`1`</sub>`+ w`<sub>`2`</sub>`x`<sub>`2`</sub>` + ... + w`<sub>`d`</sub>`x`<sub>`d`</sub>` + b`  

  - 2차 곡선 모델을 사용하면 매개변수 수가 크게 증가  
    - 매개변수 수 = `d`<sup>`2`</sup> + `d` + `1`
    - e.g., Iris 데이터 : `d` = `4` → 매개변수 수 = `21`  
    - e.g., MNIST 데이터 : `d` = `784` → 매개변수 수 = `615,441`  

<br>

#### [3] 특징 공간 변환과 표현 학습
- 선형 분리 불가능(linearly non-separable)한 원래 특징 공간
  - 직선 모델을 적용하면 75% 정확도가 한계  
  ![non_linear_feature_space][def11]  
  ![non_linear_feature_space][def12]  
  ![non_linear_feature_space][def13]  

- 표현 학습
  - 좋은 특징 공간을 자동으로 찾는 작업
  - 딥러닝은 다수의 은닉층을 가진 신경망을 이용하여 계층적인 특징 공간을 찾아냄  

<br>

- 차원에 대한 몇 가지 설명
  - 차원에 무관하게 수식은 적용 가능함  
    - e.g., `d`차원 `a`와 `b` 사이의 유클리드 거리는 모든 차원에 대해 성립  
    ![euclidean_distance][def14]  
    - 주로 2~3차원의 저차원에서 식을 고안한 후에 고차원으로 확장 적용  

  - 차원의 저주 (Curse of Dimensionality)  
    - 차원이 높아짐에 따라 데이터가 희소해지는 현상

### 3. 데이터에 대한 이해  
#### [1] 데이터 생성 과정
- 실제 기계 학습 문제는, 데이터 생성 과정을 알 수 없다.
- 단지 주어진 훈련집합으로 예측 모델 또는 생성 모델을 근사 추정할 수 있을 뿐이다.  

<br>

#### [2] 데이터베이스의 중요성
- 데이터베이스의 품질
  - 주어진 응용에 맞는 충분히 다양한 데이터를 충분한 양만큼 수집 → 추정 정확도 높아짐  
  - e.g., 정면 얼굴만 가진 데이터베이스로 학습하고 나면, 옆 얼굴은 매우 낮은 성능
  - 주어진 응용 환경을 자세히 살핀 다음 그에 맞는 데이터베이스 확보는 아주 중요함  

- 아주 많은 공개 데이터베이스
  - 기계 학습의 좋은 학습 재료로 여겨지는 3가지 데이터베이스 : Iris, MNIST, ImageNet
  - 'list of datasets for machine learning research' 위키피디아 페이지 참조
  - UCI Repository : 500개 이상의 데이터베이스가 공개되어 있음

- Iris 데이터베이스
  - 150개의 샘플, 4개의 특징, 3개의 클래스  
  ![iris_dataset][def15]  

- MNIST 데이터베이스
  - 70,000개의 샘플, 784개의 특징, 10개의 클래스  
  ![mnist_dataset][def16]  

- ImageNet 데이터베이스
  - 1,000개의 클래스, 1,400만 개 이상의 샘플  
  ![imagenet_dataset][def17]  

<br>

#### [3] 데이터베이스 크기와 기계 학습 성능  
- 데이터베이스의 왜소한 크기
  - MNIST 실험환경은 28x28 흑백 비트맵이므로, 가능한 샘플 수 `2`<sup>`784`</sup>가지이지만 MNIST 데이터베이스는 70,000개 샘플만 제공
  - 그러나 기계 학습은 70,000개 샘플로도 충분히 학습이 가능함  
  - 방대한 공간에서 실제 데이터가 발생하는 영역은 매우 작은 부분집합이므로, 실제 데이터가 발생하는 영역에서 충분한 샘플이 존재한다면 학습이 가능함

### 5. 모델 선택
#### [1] 과소적합과 과잉적합
- 아래 그림의 1차 모델은 과소적합(underfitting).  
![underfitting][def26]  

- 위 그림의 12차 모델은 과잉적합(overfitting).
  - 훈련 집합에 대해 거의 완벽하게 예측하지만, '새로운' 데이터를 예측한다면 큰 문제 발생
  - 학습 과정에서 잡음까지 모델링해버리는 현상
  - 따라서 적절한 용량의 모델을 선택하는 모델 선택 작업이 필요하다.  
  ![overfitting][def27]  

<br>

#### [2] 바이어스(bias)와 분산(variance)
- 훈련 집합을 여러 번 수집하여 1차 ~ 12차에 적용하는 실험
  - 2차 : 매번 큰 오차 → 바이어스가 큼 / 분산이 작음
  - 12차 : 매번 작은 오차 → 바이어스가 작음 / 분산이 큼
  - 바이어스와 분산은 트레이드오프(trade-off) 관계에 있음  
  ![bias_variance_tradeoff][def28]  

<br>

- 기계 학습의 목표
  - 작은 바이어스와 작은 분산을 가진 예측기(predictor) 제작이 목표
  - 따라서 바이어스 희생을 최소로 유지하며 분산을 최대로 낮추는 전략 필요  
  ![model_selection_strategy][def29]  

<br>

#### [3] 검증집합과 교차검증을 이용한 모델 선택 알고리즘
- 검증집합(validation set)을 이용한 모델 선택
  - 훈련 집합과 테스트 집합이 별도의 검증 집합을 가진 상황  
  ![validation_set][def30]  

<br>

- 교차검증(cross-validation)
  - 비용 문제로 별도의 검증집합이 없는 상황에 유용한 모델 선택 기법
  - 훈련 집합을 등분하여, 학습과 평가 과정을 여러 번 반복한 후 평균 사용  
  ![cross_validation][def31]  

<br>

- 부트스트랩(bootstrap)  
  - 난수를 이용한 샘플링 반복  
  ![bootstrap][def32]  

<br>

#### [4] 모델 선택의 한계와 현실적인 해결책
- 현실에서는 경험으로 큰 틀 선택한 후
  - 모델 선택 알고리즘으로 세부 모델 선택하는 전략 사용
  - e.g., CNN을 사용하기로 정한 후, 은닉층 개수, 활성함수, 모멘텀 계수 등을 정하는데 모델 선택 알고리즘을 적용함

- 현대 기계 학습의 전략
  - 용량이 충분히 큰 모델을 선택한 후, 선택한 모델이 정상을 벗어나지 않도록 여러 가지 **규제(regularization)** 기법을 적용함

### 7. 기계 학습 유형
#### [1] 지도 방식에 따른 유형
- 지도 학습(supervised learning)
  - 특징 벡터 `𝕏`와 목푯값 `𝕐`가 모두 주어진 상황
  - 회귀와 분류 문제로 구분

- 비지도 학습(unsupervised learning)
  - 특징 벡터 `𝕏`는 주어지는데 목푯값 `𝕐`가 주어지지 않는 상황 - PCA, Autoencoder
  - 군집화 과업 (고객 성향에 따른 맞춤 홍보 응용 등)
  - 밀도 추정, 특징 공간 변환 과업

- 강화 학습(reinforcement learning)
  - 목푯값이 주어지는데, 지도 학습과 다른 형태임
  - e.g., 바둑
    - 수를 두는 행위가 샘플인데, 게임이 끝나면 목푯값 하나가 부여됨
    - 이기면 `1`, 지면 `-1`을 부여
    - 게임을 구성한 샘플들 각각에 목푯값을 나누어 주어야 함

- 준지도 학습(semi-supervised learning)
  - 일부는 `𝕏`와 `𝕐`를 모두 가지지만, 나머지는 `𝕏`만 가진 상황 - 자가 학습(self-training)
  - 인터넷 덕분으로 `𝕏`의 수집은 쉽지만, `𝕐`는 수작업이 필요하여 최근 중요성 부각

<br>

#### [2] 다양한 기준에 따른 유형
- 오프라인 학습과 온라인 학습

- 결정론적(deterministic) 학습과 확률적(stochastic) 학습
  - 결정론적 : 같은 데이터를 가지고 다시 학습하면 같은 예측기 생성
  - 확률적 : 학습 과정에서 난수를 사용하므로 같은 데이터로 다시 학습하면 다른 예측기 생성

- 분별 모델(discriminative model)과 생성 모델(generative model)
  - 분별 모델은 부류 예측에만 관심. 즉 `P(y|x)`의 추정에 관심
  - 생성 모델은 `P(x)` 또는 `P(x|y)`를 추정함
    - 따라서 새로운 샘플을 '생성'할 수 있음
