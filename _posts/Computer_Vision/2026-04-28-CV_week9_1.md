## 특징 기술
### 1. 특징 기술자의 조건
- 특징 추출의 예
  - 필기 숫자 인식  
  ![handwritten_digit_recognition][def]  

<br>

- 특징 생성
  - 외부의 물리적 패턴을 특징 벡터(feature vector)라는 수학적 표현으로 변환  
  ![feature_vector_generation][def2]  

<br>

- 매칭이나 인식에 유용하기 위한 몇 가지 요구 조건
  - 높은 분별력(discriminating power)
  - 다양한 변환에 불변 (invariant)
    - 기하 분별성과 광도(photometric) 분별성
    - 변환에도 불구하고 같은(유사한) 값을 갖는 특징 벡터 추출해야 함  
    ![transformation_invariance][def3]  

  - 특징 벡터의 크기(차원)  
    - 차원이 낮을수록 계산이 빠름  

- 응용에 따라 공변(covariant)과 불변(invariant)을 적절히 선택해야 함
  - 매칭에서는 공변이 유용할 수 있음
  - 인식에서는 불변이 유용할 수 있음  

<br>

### 2. 관심점을 위한 기술자
- 관심점을 어떻게 기술할 것인가?
  - **윈도우의 크기**가 중요
  - 스케일 없는 관심점 (e.g., 해리스 코너)
    - 윈도우 크기를 결정하는 데 쓸 정보가 없음
    - 스케일 불변성 불가능

  - 스케일 정보가 있는 관심점 (e.g., SIFT, SURF)
    - 스케일 `σ`에 따라 윈도우 크기 결정
    - 스케일 불변성 가능

- 관심점 주위에 대한 세밀한 관찰  
![interest_point_observation][def4]  

<br>

#### [1] SIFT 기술자
- SIFT 기술자 추출 알고리즘 [Lowe 2004]  
![SIFT_algorithm][def5]  
  - 윈도우를 4x4의 16개 블록으로 분할
    - 각 블록에서 그래디언트 방향 히스토그램을 구함
    - 그래디언트 방향은 8개로 양자화
    - 그래디언트의 크기를 가중치로 사용  

  - 4x4x8 = 128차원 특징 벡터 `x`
  - `x` / `||x||`로 정규화하여 단위벡터로 변환, 광도 불변성 달성  

  - 알고리즘
    - 입력 : 입력 영상 `f`에서 검출된 키포인트 집합 `p`<sub>`i`</sub> = (`y`<sub>`i`</sub>, `x`<sub>`i`</sub>, `σ`<sub>`i`</sub>), `1` ≤ `i` ≤ `n`  

    - 출력 : 기술자가 추가된 키포인트 집합 `p`<sub>`i`</sub> = (`y`<sub>`i`</sub>, `x`<sub>`i`</sub>, `σ`<sub>`i`</sub>, `θ`<sub>`i`</sub>, `x`<sub>`i`</sub>), `1` ≤ `i` ≤ `m`  

    ```python
    for i in range(1, n+1) :
      # p_i의 그래디언트 방향 θ_i 계산
      # p_i의 특징 벡터 x_i 계산
      # x_i 정규화
    ```  

<br>

- SIFT의 변형
  - PCA-SIFG [Ke2004]
    - 키포인트에 39x39 윈도우 씌우고 도함수 `d`<sub>`y`</sub>와 `d`<sub>`x`</sub> 계산
    - 39x39x2(= 3,042)차원의 벡터를 PCA를 이용하여 20차원으로 축소

  - GLOH [Mikolajczyk2005a]
    - 원형 윈도우로 17개 영역 각각에서 16단계의 그래디언트 방향 히스토그램을 계산  
    - 17x16(= 272)차원의 벡터를 추출하고, PCA를 이용하여 128차원으로 축소

  - 모양 콘텍스트 [Mikolajczyk2005a]
    - 원형 윈도우로 36차원 특징 벡터 추출  

  ![shape_context][def6]

#### [2] Histogram of Oriented Gradients (HOG)
  - 영상을 일정 크기의 셀로 분할
  - 각 셀마다 픽셀의 그래디언트를 구해서 히스토그램으로 표현한 벡터
  - 왜곡이 적은 물체 식별에 적합  
  ![HOG][def7]  

<br>

#### [3] 이진 기술자
- 빠른 매칭을 위해 특징 벡터를 이진화하는 방법
  - 비교 쌍의 대소 관계에 따라 `0` 또는 `1`
  - 비교 쌍을 구성하는 방식에 따라 여러 변형  
  
  - 이진 기술자의 조사 패턴  
  ![binary_descriptor][def8]  

  - 매칭은 해밍 거리를 이용하여 빠르게 수행
    - 해밍 거리 : 같은 비트 수를 갖는 이진 부호 사이에 대응되는 비트가 다른 개수  
    - `11000100`과 `10000101`은 해밍 거리 `2`  

  - BRIEF, ORB, BRISK  
  ![binary_descriptor_examples][def9]
