## 매칭
### 1. 매칭의 기초
#### [1] 거리 척도
- 유클리디안 거리(Euclidean distance) vs. 마할라노비스 거리(Mahalanobis distance)
  - 확률 분포 속의 거리  
  ![euclidean_mahalanobis_distance][def]  
    - `μ`는 `b`와 `c` 중 누구와 더 가까운가?  

  - 유클리디안 거리  
  ![euclidean_distance][def2]  
    - `μ`는 `b`와 더 가까움

  - 마할라노비스 거리 : 공분산 행렬 `Σ`를 이용하여 확률 분포를 고려해 거리를 측정  
  ![mahalanobis_distance][def3]  
    - `μ`는 `c`와 더 가까움  

  - 마할라노비스 거리 예제  
  ![mahalanobis_distance_example][def4]  

  <br>

- 화이트닝 변환(whitening transformation) : 공분산 행렬이 단위 행렬 `I`가 되도록 벡터 `x`를 `y`로 변환  
  - `Σ` = `I`이면 두 거리 척도가 같음  
  ![whitening_transformation][def5]  

<br>

#### [2] 매칭 전략과 성능 분석
- 매칭을 활용하는 여러 가지 상황  
  - 파노라마 영상 제작
  - 물체 인식 또는 증강 현실  
  ![matching_applications][def6]  

- 단순한 매칭 전략
  - 두 영상의 특징 벡터를 `a`<sub>`i`</sub> (`i` = `1`, `2`, ..., `m`)와 `b`<sub>`j`</sub> (`j` = `1`, `2`, ..., `n`)라고 표기할 때,  
  아래 식을 만족하면 매칭 성공  
  ![matching_strategy][def7]  

- ROC(Receiver Operating Characteristic) 곡선을 이용한 성능 분석  

- 최근접 이웃 전략 : `a`<sub>`i`</sub>의 최근접 이웃 `b`<sub>`j`</sub>와의 거리 `d(a`<sub>`i`</sub>, `b`<sub>`j`</sub>`)`를 만족하면 매칭 성공  

- 최근접 거리 비율 전략  
  - 최근접 `b`<sub>`j`</sub>와 두 번째 최근접 `b`<sub>`k`</sub>가 아래 식을 만족하면 매칭 성공  
  ![nearest_neighbor_distance_ratio][def8]

### 2. 빠른 최근접 이웃 탐색
#### [1] *k*d 트리
- 이진검색 트리(BST, binary search tree) : 루트를 기준으로 왼쪽 부분 트리는 루트보다 작은 값, 오른쪽은 큰 값을 갖는 이진 트리  
![binary_search_tree][def9]  

- *k*d 트리 : `k`차원 공간에서의 이진 검색 트리 `X` = `{x`<sub>`1`</sub>, `x`<sub>`2`</sub>, ..., `x`<sub>`n`</sub>`}`
  - 루트 노드는 `X`를 두 개의 부분 집합 `X`<sub>`left`</sub>와 `X`<sub>`right`</sub>로 분할
  - 이 때, 분할 효과를 극대화하려면 각 차원의 분산을 계산한 후, 최대 분산을 갖는 축 `k`를 선택
  - 축을 선택했다면 `X`를 차원 `k`로 정렬하고, 결과의 중앙 값을 분할 기준으로 삼아 균형 잡힌 트리를 생성

- *k*d 트리 만들기  
![kd_tree_construction][def10]  

- 예제
  - 새로운 특징 벡터 `x` = `(7, 5.5)` 가 입력되면,
    - 루트가 `x`<sub>`2`</sub>축을 기준으로 하므로 `5.5`를 `6`과 비교하여 왼쪽으로 분기
    - `(4, 4)` 노드가 `x`<sub>`1`</sub>축을 기준으로 하므로 `7`을 `4`와 비교하여 오른쪽으로 분기
    - 반복하면 `(8, 5)` 노드에 도달  
    ![kd_tree_example][def11]
