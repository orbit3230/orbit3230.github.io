## 구조체, 열거형, 컬렉션
### 1. 구조체(Struct)
- 구조체(Struct) : 여러 값을 묶어서 사용하는 데이터 타입

```rust
struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool,
}
```

<br>

- 구조체 인스턴스 생성

```rust
let user = User {
    username: String::from("Christiano Ronaldo"),
    email: String::from("GOAT@gmail.com"),
    sign_in_count: 1,
    active: true
};
```

<br>

- 구조체 인스턴스 내부 값 변경
  - 특정 필드만 가변성을 부여하는 것은 불가능하다.  

```rust
let mut user = User {
    username: String::from("Christiano Ronaldo"),
    email: String::from("GOAT@gmail.com"),
    sign_in_count: 1,
    active: true
};

user.username = String::from("Lionel Messi");
```

- 구조체 내부 값 접근  
  - `.필드명`으로 접근 가능  

```rust
println!("Username: {}", user.username);
```

<br>

- 빠른 필드 값 설정  
  - 함수 파라미터와 필드의 이름이 일치하는 경우 생략 가능  

```rust
struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool,
}

fn build_user(username: String, email: String) -> User {
    User {
        username,
        email,
        sign_in_count: 1,
        active: true
    }
}
```

<br>

- 기존 구조체 인스턴스 값 사용  
  - 명시된 필드를 제외하고 기존 인스턴스 값 사용  

```rust
let user2 = User {
  username: String::from("Lionel Messi"),
  email: user.email,
  sign_in_count: user.sign_in_count,
  active: user.active
};
```

- 이는 아래와 같이 짧게 작성해도 같은 의미이다.  
  - `=` 연산과 동일하기 때문에, 소유권 또한 넘어간다.  

```rust
let user2 = User {
  username: String::from("Lionel Messi"),
  ..user
};
``` 

<br>

- Tuple 구조체 : 필드의 이름이 없는 구조체  
  - 동일한 필드여도 다른 타입으로 취급  

```rust
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

fn main() {
  let black = Color(0, 0, 0);
  let origin = Point(0, 0, 0);
}
```  

<br>

- 필드가 없는 구조체  
  - 데이터를 저장할 필요가 없으나 Trait를 구현할 필요가 있을 때  
  (e.g., 공통 함수 구현한 뒤 호출)  

```rust
struct AlwaysEqual;

fn main() {
  let subject = AlwaysEqual;
}
```

<br>

- 아래와 같이 구조체 인스턴스 자체를 출력하는 것은 오류를 발생한다.  

```rust
struct Rectangle {
  width: u32,
  height: u32
}
fn main() {
  let rect1 = Rectangle {
    width: 30,
    height: 50
  };
  println!("rect1 is {}", rect1);
}
```

- 하지만, **Debug Trait**를 구현하면 `{:?}`를 사용해 출력할 수 있다.  
  - Debug Trait 구현 `#[derive(Debug)]`  
  
  ```rust
  #[derive(Debug)]
  struct Rectangle {
    width: u32,
    height: u32
  }
  fn main() {
    let rect1 = Rectangle {
      width: 30,
      height: 50
    };
  }
  ```

  - `{:?}` 출력  
  
  ```rust
  println!("rect1 is {:?}", rect1);
  // rect1 is Rectangle { width: 30, height: 50 }
  ```

  - `{:#?}` 출력  
  
  ```rust
  println!("rect1 is {:#?}", rect1);
  // rect1 is Rectangle {
  //   width: 30,
  //   height: 50
  // }
  ```

  - `dbg!` 매크로  
    - 디버깅에 유용
  
  ```rust
  let scale = 2;
  let rect2 = Rectangle {
    width: dbg!(30 * scale),
    height: 50
  };
  // [src/main.rs:15] 30 * scale = 60
  ```

  <br>

- `impl` 키워드를 사용해 구조체에 메소드 추가  

```rust
#[derive(Debug)]
struct Rectangle {
  width: u32,
  height: u32
}
impl Rectangle {
  fn area(&self) -> u32 {
    self.width * self.height
  }
}

fn main() {
  let rect1 = Rectangle {
    width: 30,
    height: 50
  };
  println!(
    "The area of the rectangle is {} square pixels.",
    rect1.area()
  )
}
```

  - `&self` 는 `self: &Self`와 같은 의미이다.  
  
  ```rust
  impl Rectangle {
    fn area(self: &Self) -> u32 {
      self.width * self.height
    }
  }
  ```

- 구조체 함수 내에서 내부 값을 수정하기 위해서는 `mut` 키워드가 필요하다.  

```rust
fn add_one(&mut self) {
  self.width += 1;
}
```

- 필드 이름과 동일한 함수를 생성할 수 있다.  

```rust
impl Rectangle {
  fn width(&self) -> bool {
    self.width > 0
  }
}
```

<br>

- C나 C++와 다르게 `->` 연산자는 사용되지 않는다.  
-> Rust에서는 자동 참조 및 역참조를 통해 자동으로 처리한다.  

```rust
fn test(rect: &Rectangle) -> u32 {
  rect.width()
}
```

```rust
fn test(rect: &Rectangle) -> u32 {
  (*rect).width()
}
```

<br>

- 파라미터를 포함한 함수  

```rust
impl Rectangle {
  fn can_hold(&self, other: &Rectangle) -> bool {
    self.width > other.width && self.height > other.height
  }
}
```

<br>

- 연관 함수(Associated Function) : `impl` 블록 내에 선언된 함수  
  - 해당 타입과 연관이 있는 함수이다.  

- `self`를 파라미터로 사용하지 않는 연관 함수  
  - 주로 생성자로 사용됨  
  - 연관 함수를 호출할 때는 `::` 사용  

```rust
String::from("The function without &self");
```

```rust
/// Converts a `&str` into a [`String`].
///
/// The result is allocated on the heap.
#[inline]
fn from(s: &str) -> String {
    s.to_owned()
}
```

```rust
impl Rectangle {
  fn square(size: u32) -> Self {
    Self {
      width: size,
      height: size
    }
  }
}
```

```rust
let rect = Rectangle::square(30);
```

<br>

- 여러 개의 `impl` 블록  

```rust
impl Rectangle {
  fn area(&self) -> u32 {
    self.width * self.height
  }
}
impl Rectangle {
  fn can_hold(&self, other: &Rectangle) -> bool {
    self.width > other.width && self.height > other.height
  }
}
```

### 2. 열거형(Enum)  
- 여러 값 중 하나를 나타내는 자료형  

```rust
enum IpAddrKind {
  V4,
  V6
}
fn main() {
  let four = IpAddrKind::V4;
  let six = IpAddrKind::V6;
}
```

- 함수 사용 예시

```rust
fn route(ip_type: IpAddrKind) {}
fn main() {
  route(IpAddrKind::V4);
  route(IpAddrKind::V6);
}
```

- 구조체 + 열거형 사용 예시  

```rust
enum IpAddrKind {
  V4,
  V6
}
struct IpAddr {
  kind: IpAddrKind,
  address: String
}
fn main() {
  let home = IpAddr {
    kind: IpAddrKind::V4,
    address: String::from("127.0.0.1")
  }
  let loopback = IpAddr {
    kind: IpAddrKind::V6,
    address: String::from("::1")
  }
}
```

<br>

- 값을 가지는 열거형

```rust
enum IpAddr {
  V4(String),
  V6(String)
}
fn main() {
  let home = IpAddr::V4(String::from("127.0.0.1"));
  let loopback = IpAddr::V6(String::from("::1"));
}
```

- 다른 타입의 값을 가질 수도 있다.  

```rust
enum IpAddr {
  V4(u8, u8, u8, u8),
  V6(String)
}
fn main() {
  let home = IpAddr::V4(127, 0, 0, 1);
  let loopback = IpAddr::V6(String::from("::1"));
}
```

- 다양한 타입 포함 가능  

```rust
enum Message {
  Quit,
  Move { x: i32, y: i32 },
  Write(String),
  ChangeColor(i32, i32, i32)
}
```

<br>

- 열거형 함수 작성

```rust
enum Message {
  Quit,
  Move { x: i32, y: i32 },
  Write(String),
  ChangeColor(i32, i32, i32)
}
impl Message {
  fn call(&self) {}
}
fn main() {
  let m = Message::Write(String::from("hello"));
  m.call();
}
```

<br>

- **Option Enum** : 값의 유무 여부가 확실하지 않을 경우 사용  

```rust
fn get_value() -> Option<i32> {
  let should_return = true;
  if should_return {
    Some(5)
  } else {
    None
  }
}
fn main() {
  let some_number: Option<i32> = get_value();
}
```

- Option 열거형은 바로 사용 불가  

```rust
let x: i8 = 5;
let y = Option<i8> = Some(5);

let sum = x + y;  // no implementation for `i8 + Option<i8>` Error  
```

<br>

- Match 제어 흐름 구조  

```rust
enum Coin {
  Penny,
  Nickel,
  Dime,
  Quarter
}
fn value_in_cents(coin: Coin) -> u8 {
  match coin {
    Coin::Penny => {
      println!("Lucky penny!");
      1
    }
    Coin::Nickel => 5,
    Coin::Dime => 10,
    Coin::Quarter => 25
  }
}
```

- 사용 예시  

```rust
fn plus_one(x: Option<i32>) -> Option<i32> {
  match x {
    None => None,  // 모든 경우의 수를 처리하기 위해 반드시 필요  
    Some(i) => Some(i + 1)
  }
}
fn main() {
  let five = Some(5);
  let six = plus_one(five);
  let none = plus_one(None);
}
```

- `other`로 나머지 처리 가능  

```rust
enum Message {
  Quit,
  Move { x: i32, y: i32 },
  Write(String),
  ChangeColor(i32, i32, i32)
}
fn handle_message(message: Message) {
  match message {
    Meesage::Quit => {},
    other => {}
  }
}
```

- 값이 필요 없는 경우 `_` 사용  

```rust
fn handle_message(message: Message) {
  match message {
    Meesage::Quit => {},
    _ => {}
  }
}
```

- 특정 값에서만 작동하는 코드 만들기  

```rust
fn main() {
  let config_max = Some(3u8);
  match config_max {
    Some(max) => println!("The maximum is configured to be {}", max),
    _ => (),  // 다른 경우는 무시
  }
}
```

```rust
fn main() {
  let config_max = Some(3u8);
  if let Some(max) = config_max {
    println!("The maximum is configured to be {}", max);
  } else {
    println!("Input value is not valid");
  }
}
```

<br>

- Enum 사용 예  

```rust
#[derive(Debug)]
enum Message {
  Quit,
  Move { x: i32, y: i32 },
  ChangeColor(i32, i32, i32)
}

fn main() {
  let msgs = {
    Message::Quit,
    Message::Move { x: 1, y: 3 },
    Message::ChangeColor(255, 255, 0)
  };
  for msg in msgs {
    println!("{:?}", msg);
  }
}
```

```rust
fn plus_one(x: Option<i32>) -> Option<i32> {
  match x {
    None => None,
    Some(i) => Some(i + 1)
  }
}

fn main() {
  let five = Some(5);
  let six = plus_one(five);
  let none = plus_one(None);

  assert_eq!(six, Some(6));
  assert_eq!(none, None);
}
```

### 3. 컬렉션(Collection)
#### [1] Vector
- 단일 데이터 타입을 여러 개 저장하는 자료구조  
  - Heap 영역에 데이터 저장
  - 빈 벡터를 생성하는 경우 타입 정보 필요  

  ```rust
  let v: Vec<i32> = Vec::new();
  ```

  - 기본적으로는 값에서 타입 추론 가능  

  ```rust
  let v = vec![1, 2, 3];
  ```

  <br>

- 벡터에 값 추가하기  
  - 값을 추가하는 경우에도 타입 추론

```rust
let mut v = Vec::new();

v.push(5);
v.push(6);
v.push(7);
v.push(8);
```

<br>

- 벡터 요소 접근  

```rust
let v = vec![1, 2, 3, 4, 5];

let third: &i32 = &v[2];  // 인덱스 접근의 경우 오류 발생 가능성 있음
println!("The third element is {}", third);

let fourth: Option<&i32> = v.get(3);
match fourth {
  Some(fourth) => println!("The fourth element is {}", fourth),
  None => println!("There is no fourth element")
}
```

- 아래 코드는 오류 발생  

```rust
let mut v = vec![1, 2, 3, 4, 5];
let first = &v[0];  // 불변 참조자 생성
v.push(6);  // 가변 참조자 생성성
println!("The first element is: {}", first);  // 불변 참조자 사용
```

<br>

- 벡터 값 루프

```rust
fn main() {
  let v = vec![100, 32, 57];
  for i in &v {
    println!("{i}");
  }
}
```

```rust
fn main() {
  let mut v = vec![100, 32, 57];
  for i in &mut v {
    *i += 50;
  }
}
```

<br>

- 열거형을 사용하여 여러 타입의 데이터 저장 가능  

```rust
enum SpreadsheetCell {
  Int(i32),
  Float(f64),
  Text(String)
}
fn main() {
  let row = vec![
    SpreadsheetCell::Int(3),
    SpreadsheetCell::Text(String::from("blue")),
    SpreadsheetCell::Float(10.12)
  ];
}
```

<br>

- 벡터가 스코프에서 벗어나는 경우, 모든 메모리 할당 해제. 이후 내용 참조 불가능  

```rust
fn main() {
  {
    let v = vec![1, 2, 3, 4];
  }
  // v는 여기서 사용 불가능
}
```

<br>

- Vector 사용 예  

```rust
fn main() {
  let v = vec![0, 1, 3, 6, 10];
  let mut sum = 0;
  for i in 0..5 {
    sum += i;
    assert_eq!(v[i], sum);
  }
}
```

```rust
fn main() {
  let v = vec![1, 2];
  let first = &v[0];
  let second = v.get(1);
  let third = v.get(2);

  assert_eq!(first, &1);
  assert_eq!(second, Some(&2));
  assert_eq!(third, None);
}
```

#### [2] String
- UTF-8로 인코딩된 문자열 타입
- 벡터에서 일부 기능을 추가하여 구현했기 때문에, 대부분의 함수를 그대로 사용 가능하다.  

<br>

- 문자열을 생성하는 다양한 방법  

```rust
let mut s = String::new();
```

```rust
let s = "initial contents".to_string();
```

```rust
let s = String::from("initial contents");
```

<br>

- 문자열 업데이트  

```rust
let mut s = String::from("foo");
s.push_str("bar");  // push_str 에서는 소유권을 이동하지 않는다.  
s.push("O");
let s1 = String::from("푸바오야 ");
let s2 = String::from("가지마ㅠㅠ");
let s3 = s1 + &s2;  // s1은 이후 사용 불가능
// + 연산자는 내부적으로 push_str로 구현되어 있음
```
